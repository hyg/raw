## 文件命名规范

| 文件类型 | 命名格式 | 说明 |
|----------|----------|------|
| 每日摄入日志 | `d.yyyymmdd.yaml` | `yyyy`=年，`mm`=月，`dd`=日 |
| 食材营养数据 | `e.xxx.yaml` | `xxx`=食材名称（可含版本号） |
| 配餐目标 | `NRV.xxx.yaml` | `xxx`=目标名称（热量值或版本号） |

### 食材命名说明

- **基础食材**：`e.大米.yaml`、`e.菠菜.yaml`、`e.鸡蛋.yaml`
- **带版本号的食材**：`e.馒头 2603a.yaml`、`e.炖羊蝎子 2603a.yaml`
  - 版本号格式：`YYYYMMx`，表示不同批次或配方
- **补充剂**：`e.selenium.yaml`、`e.mature multi.yaml`

### 配餐目标命名说明

- **热量命名**：`NRV.1800.yaml`、`NRV.2000.yaml` - 数字表示目标热量（kcal）
- **版本命名**：`NRV.202405a.yaml`、`NRV.202409b.yaml` - `YYYYMMx` 格式表示不同版本

## 每日摄入日志

**文件位置**: `food/d.yyyymmdd.yaml`

### 结构

```yaml
date: <整数>              # 日期 YYYYMMDD
food:                     # 食物摄入记录（必需）
  - time: <整数>          # 时间 YYYYMMDDHHMMSS
    name: <字符串>        # 食物名称（引用 e.xxx.yaml）
    amount: <数字>        # 摄入量
    unit: <字符串>        # 单位
water:                    # 饮水记录（可选）
  - time: <整数>
    name: <字符串>        # 饮品名称
    amount: <数字>
    unit: ml
med:                      # 补充剂记录（可选）
  - time: <整数>
    name: <字符串>        # 补充剂名称
    amount: <数字>
    unit: <字符串>        # g/μg/mg/片
log: <字符串>             # 日志文本（可选）
comment: <字符串>         # 评论文本（可选）
```

### 字段说明

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `date` | 整数 | 是 | 日期，格式 YYYYMMDD |
| `food` | 数组 | 是 | 食物摄入记录列表 |
| `water` | 数组 | 否 | 饮水记录列表 |
| `med` | 数组 | 否 | 补充剂/药物记录列表 |
| `log` | 字符串 | 否 | 当日日志备注 |
| `comment` | 字符串 | 否 | 当日评论 |

## 食材营养数据

**文件位置**: `food/e.xxx.yaml`

### 类型一：基础食材

```yaml
name: <字符串>            # 食材名称
amount: <数字>            # 基准量（通常为 100）
unit: g                   # 单位（通常为 g）
element:                  # 营养成分表
  热量:
    amount: <数字>
    unit: kj/kcal
    nrv: <数字>           # NRV 百分比
  蛋白质:
    amount: <数字>
    unit: g
    nrv: <数字>
  # ... 其他营养成分
```

### 类型二：复合食材

```yaml
name: <字符串>            # 食材名称
amount: <数字>            # 总量
unit: g                   # 单位
food:                     # 原料列表
  - name: <字符串>
    amount: <数字>
    unit: g
log:                      # 制作日志（可选）
  - <字符串>
comment:                  # 评论（可选，数组类型）
  - <字符串>
```

### 类型三：补充剂

```yaml
name: <字符串>            # 补充剂名称
amount: <数字>            # 基准量
unit: μg/mg/片            # 单位
element:                  # 有效成分
  硒:
    amount: <数字>
    unit: μg
    nrv: <数字>
```

## 配餐目标

**文件位置**: `food/NRV.xxx.yaml`

### 结构

```yaml
name: <字符串>            # 配餐目标名称
DRIs: <字符串>            # 膳食营养素参考摄入量标准版本
element:                  # 营养素目标值
  热量:
    amount: <数字>
    unit: kcal
  蛋白质:
    amount: <数字>
    unit: g
  脂肪:
    amount: <数字>
    unit: g
  碳水化合物:
    amount: <数字>
    unit: g
  水:
    amount: <数字>
    unit: ml
```

### 字段说明

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `name` | 字符串 | 是 | 配餐目标名称 |
| `DRIs` | 字符串 | 是 | 采用的标准版本，如 `DRIforChina2023` |
| `element` | 对象 | 是 | 营养素目标值对象 |

### 核心营养素

所有配餐目标文件必须包含以下 5 种核心营养素：

| 营养素 | 单位 | 说明 |
|--------|------|------|
| 热量 | kcal | 每日能量目标 |
| 蛋白质 | g | 每日蛋白质目标 |
| 脂肪 | g | 每日脂肪目标 |
| 碳水化合物 | g | 每日碳水化合物目标 |
| 水 | ml | 每日水摄入目标 |

### 示例配餐目标

| 目标名称 | 热量 (kcal) | 蛋白质 (g) | 脂肪 (g) | 碳水 (g) | 水 (ml) |
|----------|-------------|------------|----------|----------|---------|
| 1750 | 1750 | 76.56 | 43.75 | 251.56 | 2000 |
| 1800 | 1800 | 78.75 | 50.00 | 258.75 | 2000 |
| 2000 | 2000 | 87.50 | 55.56 | 287.50 | 2000 |
| 2050 | 2050 | 89.69 | 56.94 | 294.69 | 2000 |

## 数据类型定义

### 时间格式

- **日期**: `YYYYMMDD`（8 位整数），如 `20260323`
- **时间戳**: `YYYYMMDDHHMMSS`（14 位整数），如 `20260323113400`

### 单位枚举

**摄入量单位**:
- `g` - 克
- `kg` - 千克
- `ml` - 毫升
- `L` - 升
- `个` - 个
- `片` - 片
- `碗` - 碗
- `杯` - 杯
- `勺` - 勺

**营养成分单位**:
- `kj` - 千焦
- `kcal` - 千卡
- `g` - 克
- `mg` - 毫克
- `μg` - 微克

## 示例

### 每日摄入日志示例

```yaml
date: 20260323
food:
  - time: 20260323113400
    name: 柑橘
    amount: 104.0
    unit: g
  - time: 20260323113800
    name: 馒头 2603a
    amount: 1
    unit: 个
water:
  - time: 20260323095000
    name: 开水
    amount: 200
    unit: ml
med:
  - time: 20260323125100
    name: selenium
    amount: 200
    unit: μg
log: |
comment: |
```

### 基础食材示例

```yaml
name: 大米
amount: 100
unit: g
element:
  热量:
    amount: 1467
    unit: kj
    nrv: 17
  蛋白质:
    amount: 6.3
    unit: g
    nrv: 11
  脂肪:
    amount: 1.2
    unit: g
    nrv: 2
  碳水化合物:
    amount: 77.4
    unit: g
    nrv: 26
```

### 复合食材示例

```yaml
name: 炖羊蝎子 2603a
amount: 371.0
unit: g
food:
  - name: 羊肉
    amount: 439.5
    unit: g
  - name: 双笑咸山黄皮果
    amount: 50.0
    unit: g
log:
  - 羊蝎子带骨生重 480.0 克，带骨熟重 411.5 克.
  - 羊蝎子加山黄皮在高压锅煮 40 分钟。
comment:
  - 足够软烂，筋和骨髓都能吸到嘴里。
  - 第一餐带骨熟重 113.5 克、骨重 40.5 克、肉重 73.0 克。
```

### 补充剂示例

```yaml
name: mature multi
amount: 1

element:
  VA(视黄醇等):

    unit: μg
    nrv: 83
  VC(抗坏血酸):
    amount: 90
    unit: mg
    nrv: 100
  锌:
    amount: 11
    unit: mg
    nrv: 100
```
